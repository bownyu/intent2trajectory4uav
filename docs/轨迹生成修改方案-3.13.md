## 1. 设计目标

新生成器必须满足五个目标：
1. **标签目标从几何分类转为站点相对风险分类**
2. **同一意图内存在多风格、多机型、多速度结构**
3. **所有样本都能映射到可计算的风险语义向量**
4. **所有样本都通过机型相关的运动学检查**
5. **数据集具备可控多样性，避免后续训练过拟合**
    

---

## 2. 总体标签体系

每条样本必须同时具有 5 层描述：
primary_intent   : attack | retreat | hover | loiter  
motion_style     : 该意图下的二级风格  
airframe_profile : 机型/动力学配置  
risk_vector      : 6维连续语义分数  
stage_plan       : 由哪些行为阶段拼接而成
### 2.1 一级标签
attack  
retreat  
hover  
loiter
### 2.2 二级风格（建议）

attack:  
  direct_commit  
  weave_commit  
  climb_dive_commit  
  turn_dive_commit  
  zigzag_commit  
  probe_recommit  
  
retreat:  
  direct_breakaway  
  arc_breakaway  
  zigzag_breakaway  
  climb_breakaway  
  
hover:  
  true_hover  
  drift_hold  
  corridor_hold  
  pseudo_hover_racetrack  
  
loiter:  
  circle_loiter  
  ellipse_loiter  
  petal_loiter  
  racetrack_loiter  
  probe_loiter

说明：

- `figure8_loiter` 不建议作为默认主风格名。  
    因为“8字”是几何语言，不是风险语言，而且相对站点的 bearing 行为有时会很混，建议用 `petal_loiter` 或 `racetrack_loiter` 替代。
    
- `hover` 内部必须允许 `pseudo_hover_racetrack`，否则固定翼无法生成物理可行样本。

## 3. 坐标、状态与基础定义

站点位于局部 ENU 坐标系原点。

对第 `i` 个时刻，定义：

p_i   = [x_i, y_i]                # 水平位置  
r_i   = ||p_i||                   # 相对站点水平距离  
phi_i = unwrap(atan2(y_i, x_i))   # 相对站点方位角（展开后）  
v_i   = [vx_i, vy_i]              # 水平速度  
s_i   = ||v_i||                   # 水平速度大小

定义单位向量：

e_r_i     = p_i / (r_i + eps)             # 由站点指向无人机的径向单位向量（向外）  
e_theta_i = [-e_r_i.y, e_r_i.x]           # 切向单位向量

定义径向/切向速度：

v_r_i     = dot(v_i, e_r_i)               # 正值表示远离站点，负值表示逼近站点  
v_theta_i = dot(v_i, e_theta_i)

定义速度航迹角与机体航向：

chi_i = atan2(vy_i, vx_i)                 # course / 速度方向  
psi_i = yaw_i                             # 机体朝向  
psi_to_station_i = atan2(-y_i, -x_i)      # 指向站点的方位

定义两个“指向站点”对齐度：

align_course_i = cos(wrap(chi_i - psi_to_station_i))  
align_yaw_i    = cos(wrap(psi_i - psi_to_station_i))

说明：

- `course` 反映“飞行方向是否朝向站点”
    
- `yaw` 反映“机体/传感器是否面向站点”
    
- 风险语义里的“指向性”必须同时考虑这两者
    

---

## 4. 风险距离带配置

必须显式配置站点风险带，不能只用一个半径。

建议配置：

R_core                 # 核心禁入半径  
R_sensitive            # 敏感带外半径  
B_hold   = [R_hold_in,   R_hold_out]      # 对峙/占位带  
B_loiter = [R_loiter_in, R_loiter_out]    # 侦察/盘旋带  
R_outer_start          # 远距生成起点参考半径  
R_exit                 # 撤离充分外扩半径

约束关系建议：

R_core < R_sensitive < R_hold_in <= R_hold_out < R_outer_start < R_exit  
R_core < R_loiter_in < R_loiter_out < R_hold_out 或按任务单独定义

每个样本还应有一个 `active_band`：

- `hover` 的 `active_band = B_hold`
    
- `loiter` 的 `active_band = B_loiter`
    
- `attack` 可选 `B_terminal = [R_core, R_sensitive]`
    
- `retreat` 通常不需要长期活动带，但可记录 `escape_origin_band`

## 5. 六维风险语义向量

记风险向量为：

risk_vector = [  
  close_score,      # 接近性  
  dwell_score,      # 滞留性  
  encircle_score,   # 环绕性  
  point_score,      # 指向性  
  uncertain_score,  # 不确定性  
  disengage_score   # 脱离性  
]

所有维度归一化到 `[0, 1]`。

---

### 5.1 接近性 `close_score`

由三部分组成：

close_frac     = mean(v_r_i < -v_close_thr)  
net_close      = clip((r_0 - r_T) / max(r_0 - R_core, eps), 0, 1)  
intrusion_depth= clip((R_sensitive - min(r_i)) / max(R_sensitive - R_core, eps), 0, 1)

推荐：

close_score = 0.45 * close_frac  
            + 0.35 * net_close  
            + 0.20 * intrusion_depth

含义：

- 是否大部分时间都在逼近
    
- 最终是否明显更接近
    
- 是否真正压入敏感带
    

---

### 5.2 滞留性 `dwell_score`

相对于 `active_band` 计算：

dwell_score = mean(r_i in active_band)

`hover` / `loiter` 主要使用这个指标。  
`attack` 一般不会把它作为主判据，但可作为辅因子。

---

### 5.3 环绕性 `encircle_score`

先计算累计方位变化：

bearing_cum_change = sum(abs(wrap(phi_i - phi_{i-1})))  
encircle_cycles    = bearing_cum_change / (2*pi)

再归一化：

encircle_score = clip(encircle_cycles / C_ref, 0, 1)

默认可取：

C_ref = 1.0

也可以按样本时长配置 `C_ref(duration)`。

---

### 5.4 指向性 `point_score`

必须同时看速度方向与机体朝向。

course_point_ratio = mean(align_course_i > cos(alpha_course))  
yaw_point_ratio    = mean(align_yaw_i    > cos(alpha_yaw))

推荐：

point_score = w_course * course_point_ratio  
            + w_yaw    * yaw_point_ratio

默认：

alpha_course = 30 deg  
alpha_yaw    = 35 deg

默认权重建议：

- `attack`: `w_course = 0.7`, `w_yaw = 0.3`
    
- `hover/loiter`: `w_course = 0.3`, `w_yaw = 0.7`
    
- `retreat`: `w_course = 0.6`, `w_yaw = 0.4`
    

---

### 5.5 不确定性 `uncertain_score`

这不是随机噪声，而是“试探、假动作、诱骗”的结构化指标。

由三部分组成：

sign_changes   = count_sign_change(smooth(v_r_i), deadband=v_r_eps)  
abort_count    = count_abort_events(r_i)  
path_ratio     = path_length / max(||p_T - p_0||, eps)

推荐：

uncertain_score = clip(  
    0.40 * min(sign_changes / N_sign_ref, 1.0) +  
    0.40 * min(abort_count  / N_abort_ref, 1.0) +  
    0.20 * clip((path_ratio - 1.0) / (rho_ref - 1.0), 0, 1),  
0, 1)

建议默认：

N_sign_ref  = 4  
N_abort_ref = 2  
rho_ref     = 1.8

#### `abort_count` 检测算法

一次 `abort` 定义为：

1. 距离先显著减小：`r_before - r_min >= Δr_probe`
    
2. 随后在短窗口内显著回拉：`r_after - r_min >= Δr_abort`
    
3. 之后再次发生显著内压：`r_after - r_recommit >= Δr_recommit`
    

推荐默认：

Δr_probe     = 80 m  
Δr_abort     = 60 m  
Δr_recommit  = 60 m  
W_abort      = 5~20 s

---

### 5.6 脱离性 `disengage_score`

open_frac = mean(v_r_i > v_open_thr)  
net_open  = clip((r_T - r_0) / max(R_exit - r_0, eps), 0, 1)

推荐：

disengage_score = 0.55 * open_frac + 0.45 * net_open

## 6. 四类意图评分与判定逻辑

### 6.1 必须采用“双层判定”

每个样本生成后必须经过：

1. **连续评分**：计算 `I_attack / I_retreat / I_hover / I_loiter`
    
2. **硬阈值判定**：检查目标类是否达到门槛
    
3. **模糊性排斥**：目标类分数必须明显高于第二名
    

即：

accept if:  
  I_target >= threshold_target  
  and I_target - second_best >= margin_min

推荐默认：

threshold_target = 0.70 ~ 0.75  
margin_min       = 0.10 ~ 0.15

---

### 6.2 `attack` 评分

I_attack = 0.35 * close_score  
         + 0.25 * point_score  
         + 0.15 * (1 - disengage_score)  
         + 0.15 * intrusion_depth  
         + 0.10 * uncertain_score

硬约束建议：

close_frac       >= 0.55  
net_close        >= 0.30  
point_score      >= 0.50  
disengage_score  <= 0.35

说明：

- `uncertain_score` 可高可低，不是攻击必要条件
    
- 但 `probe_recommit` / `feint_attack` 风格必须要求 `uncertain_score` 达到更高阈值
    

---

### 6.3 `retreat` 评分

定义：

outward_monotonic_ratio = mean(r_i - r_{i-1} >= -dr_tol)

评分：

I_retreat = 0.40 * disengage_score  
          + 0.25 * outward_monotonic_ratio  
          + 0.20 * (1 - close_score)  
          + 0.15 * (1 - point_score)

硬约束建议：

disengage_score        >= 0.65  
outward_monotonic_ratio>= 0.80  
r_T >= max(1.5 * r_0, r_0 + Δr_min)

默认：

Δr_min = 300 m 或按任务配置

---

### 6.4 `hover` 评分（内部语义：`standoff_hold`）

先定义：

dwell_hold       = dwell_ratio(B_hold)  
radius_stability = 1 - clip(std(r_i) / (0.5 * width(B_hold)), 0, 1)  
radial_drift     = abs(r_T - r_0) / max(width(B_hold), eps)  
encircle_penalty = clip(encircle_cycles / C_hover_max, 0, 1)

评分：

I_hover = 0.35 * dwell_hold  
        + 0.25 * radius_stability  
        + 0.20 * (1 - encircle_penalty)  
        + 0.20 * (1 - radial_drift)

硬约束建议：

dwell_hold      >= 0.70  
close_score     <= 0.25  
disengage_score <= 0.25  
encircle_cycles <= C_hover_max_family

这里 `C_hover_max_family` 必须随机型变化：

- 多旋翼：`0.25 ~ 0.40`
    
- 固定翼：`0.50 ~ 0.80`
    
- VTOL：按当前模式取值
    

这点非常重要，否则固定翼的“占位监视”会被错误排斥。

---

### 6.5 `loiter` 评分

先定义：

dwell_loiter         = dwell_ratio(B_loiter)  
tangential_dominance = mean(abs(v_theta_i)) / (mean(abs(v_r_i)) + eps)  
radial_neutrality    = 1 - clip(abs(r_T - r_0) / max(width(B_loiter), eps), 0, 1)  
tan_dom_score        = clip(tangential_dominance / TD_ref, 0, 1)

推荐：

TD_ref = 1.5

评分：

I_loiter = 0.30 * dwell_loiter  
         + 0.30 * encircle_score  
         + 0.20 * tan_dom_score  
         + 0.20 * radial_neutrality

硬约束建议：

dwell_loiter         >= 0.65  
encircle_cycles      >= 0.75  
tangential_dominance >= 1.50  
close_score          <= 0.45  
disengage_score      <= 0.35

说明：

- `probe_loiter` 允许短时 inward probe
    
- 但不能转化成持续 `attack`
    
- 即：允许局部短窗口 `close_score` 上升，但全局不能跨越 `attack` 判定线

## 8. 机型联动设计

---

### 8.1 机型配置结构

@dataclass  
class AirframeProfile:  
    name: str  
    family: Literal["multirotor", "fixed_wing", "vtol"]  
  
    hover_capable: bool  
    v_min: float  
    v_cruise_range: tuple[float, float]  
    v_dash_range: tuple[float, float]  
  
    a_long_max: float  
    a_brake_max: float  
    a_lat_max: float  
  
    climb_rate_max: float  
    descent_rate_max: float  
    jerk_max: float  
  
    yaw_rate_max: float  
    turn_rate_max: float  
    min_turn_radius: float  
  
    control_tau: float  
    preferred_altitude: tuple[float, float]  
  
    allowed_styles: dict[str, list[str]]  
    validator_overrides: dict[str, float]

---

### 8.2 机型不是修饰项，而是主导约束项

同一意图的实现必须由机型决定。

#### `hover`

- 多旋翼：`true_hover / drift_hold`
    
- 固定翼：`corridor_hold / pseudo_hover_racetrack`
    
- VTOL：可在 `hover` 模式与 `cruise` 模式之间切换
    

#### `loiter`

- 多旋翼：可小半径圆/椭圆/花瓣
    
- 固定翼：更适合大半径 orbit / racetrack
    
- VTOL：两者都可
    

#### `attack`

- 多旋翼：低速逼近、试探、短停顿后再压
    
- 固定翼：高速压入、弧形切入、转向后压、掠袭式逼近
    
- VTOL：取决于当前阶段的 flight mode
    

#### `retreat`

- 多旋翼：直接拉开、短折返后外逃、爬升脱离
    
- 固定翼：更偏持续外逃与弧线外摆
    

---

### 8.3 关键原则：语义相同，不等于形态相同

例如 `hover`：

- 多旋翼的 `hover` 可能真的是近定点悬停
    
- 固定翼的 `hover` 只能是**相对站点语义上的占位保持**
    

因此内部语义必须叫 `standoff_hold`，而不是“绝对速度接近零”。

## 9. 生成器核心架构：从“画轨迹”改成“阶段指令 + 动力学展开”

---

### 9.1 统一行为阶段 `Stage`

每个样本由若干 `Stage` 拼接而成。  
每个阶段不是直接给 `x,y,z`，而是给出**相对站点的期望运动指令**。

@dataclass  
class StageSpec:  
    name: str  
    duration_range: tuple[float, float]  
    terminate_rule: str  
  
    vr_cmd: dict       # 径向速度指令，正外逃，负逼近  
    vt_cmd: dict       # 切向速度指令  
    vz_cmd: dict       # 垂向速度指令  
  
    yaw_mode: str      # course_locked / station_facing / station_scan / decoupled  
    noise_profile: str  
    semantic_effects: dict

---

### 9.2 指令空间统一为站点极坐标下的速度命令

每个阶段输出：

v_r_cmd(t)  
v_theta_cmd(t)  
v_z_cmd(t)  
yaw_mode(t)

再由动力学模型积分成实际轨迹。

这是最关键的改造点，因为它天然对齐你的六维语义：

- `v_r_cmd` 决定接近/脱离
    
- `v_theta_cmd` 决定环绕
    
- `active_band` + `v_r_cmd` 决定滞留
    
- `yaw_mode` + `chi` 决定指向性
    
- `符号切换/中断/回拉` 决定不确定性
    

---

### 9.3 基础指令模板

统一写成：

v_r_cmd(t)     = base_r + A_r * sin(2π t / T_r + φ_r) + envelope_r(t)  
v_theta_cmd(t) = base_t + A_t * sin(2π t / T_t + φ_t) + envelope_t(t)  
v_z_cmd(t)     = base_z + A_z * sin(2π t / T_z + φ_z) + envelope_z(t)

其中 `envelope_*` 可取：

- constant
    
- ramp_up
    
- ramp_down
    
- bell
    
- triangular
    
- piecewise_constant
    

这样程序不需要“硬编码很多轨迹”，而是通过少量可组合的指令构件生成大量行为。

## 10. 四类意图的阶段语法

---

### 10.1 `attack`

#### 公共定义

`attack` 的主目标不是“到原点的几何路径”，而是：

- `close_score` 高
    
- `point_score` 高
    
- `disengage_score` 低
    
- 可伴随中等 `uncertain_score`
    

#### 推荐风格与阶段

##### 1) `direct_commit`

close_in         -> terminal_commit

建议参数：

- `close_in`: `v_r_cmd = -(0.4~0.7) * v_cruise`
    
- `terminal_commit`: `v_r_cmd = -(0.7~1.0) * v_dash`
    
- `v_theta_cmd` 小且后段衰减
    
- `yaw_mode = course_locked` 或 `station_facing`
    

硬约束：

- `close_score >= 0.75`
    
- `point_score >= 0.65`
    
- `uncertain_score <= 0.30`
    

---

##### 2) `weave_commit`

close_in_with_weave -> terminal_commit

建议参数：

- `v_r_cmd < 0`
    
- `v_theta_cmd = bias + A*sin(...)`
    
- `A = 0.15~0.45 * v_cruise`
    
- 周期 `T = 8~30 s`
    

硬约束：

- `path_ratio >= 1.08`
    
- `sign_changes(v_theta) >= 2` 或横向摆动量足够
    
- `close_score >= 0.65`
    

---

##### 3) `climb_dive_commit`

climb_close -> dive_commit

建议参数：

- 前段 `v_z_cmd > 0`
    
- 后段 `v_z_cmd < 0`
    
- `v_r_cmd` 全程为负
    
- `yaw_mode = course_locked`
    

硬约束：

- `Δz_up >= H_up_min`
    
- `Δz_down >= H_down_min`
    
- `point_score >= 0.55`
    

---

##### 4) `turn_dive_commit`

offset_turn -> recommit_dive

建议参数：

- 中前段单侧 `v_theta_cmd` 偏大，符号固定
    
- 后段 `v_theta_cmd` 逐步衰减到 0
    
- `v_r_cmd` 后段明显增大为更强负值
    

硬约束：

- 有明显单侧 heading excursion
    
- `close_score >= 0.60`
    
- `encircle_score` 不高于 `loiter` 区间
    

---

##### 5) `zigzag_commit`

close_in -> segmented_zigzag -> terminal_commit

建议参数：

- `v_theta_cmd` 分段交替正负
    
- 交替次数 `2~5`
    

硬约束：

- `sign_changes(v_theta) >= 2`
    
- `close_score >= 0.60`
    

---

##### 6) `probe_recommit`

probe_close -> abort_offset -> recommit -> terminal_commit

建议参数：

- `probe_close`: `v_r_cmd < 0`
    
- `abort_offset`: 短时间 `v_r_cmd > 0` 或 `v_theta_cmd` 大幅偏移
    
- `recommit`: 再次 `v_r_cmd < 0`
    
- `yaw_mode` 可切为 `station_scan`
    

硬约束：

- `abort_count >= 1`
    
- `uncertain_score >= 0.35`
    
- `close_score >= 0.60`
    
- `disengage_score` 不能高到变成 `retreat`
    

---

### 10.2 `retreat`

#### 公共定义

核心不是“远离原点的直线”，而是：

- `disengage_score` 高
    
- `outward_monotonic_ratio` 高
    
- `close_score` 低
    

#### 风格

##### 1) `direct_breakaway`

breakaway -> extend_out

参数：

- `v_r_cmd = +(0.5~0.9) * v_cruise`
    
- `v_theta_cmd` 小
    
- 高度可稳定或小波动
    

---

##### 2) `arc_breakaway`

breakaway_with_arc -> extend_out

参数：

- `v_r_cmd > 0`
    
- `v_theta_cmd` 单侧偏置，符号固定
    
- 形成单边外摆
    

硬约束：

- heading excursion 达到最小值
    
- 横向符号变化不能过多
    

---

##### 3) `zigzag_breakaway`

breakaway -> zigzag_extend

参数：

- `v_theta_cmd` 分段正负交替
    
- `v_r_cmd` 始终保持总体为正
    

硬约束：

- `sign_changes(v_theta) >= 2`
    
- `disengage_score >= 0.65`
    

---

##### 4) `climb_breakaway`

breakaway -> climb_extend

参数：

- `v_r_cmd > 0`
    
- `v_z_cmd > 0` 持续一段时间
    

硬约束：

- `Δz >= H_climb_min`
    

---

### 10.3 `hover`（内部语义：`standoff_hold`）

#### 公共定义

核心不是“速度接近 0”，而是：

- 高 `dwell_score`
    
- 低 `encircle_score`
    
- 低净接近、低净脱离
    
- 半径稳定
    

#### 风格

##### 1) `true_hover`

仅 hover-capable 机型可用。

settle_to_band -> hover_hold

参数：

- `v_r_cmd = k_r * (r_target - r)`
    
- `v_theta_cmd` 极小
    
- `v_z_cmd` 小幅波动
    
- `yaw_mode = station_facing` 或 `station_scan`
    

---

##### 2) `drift_hold`

settle_to_band -> drift_hold

参数：

- 在固定主轴上做低频来回漂移
    
- 切向运动小
    
- 半径变化小
    

---

##### 3) `corridor_hold`

适合固定翼和高速平台。

enter_hold_corridor -> back_and_forth_hold

参数：

- 在一个局部 corridor 内往返
    
- corridor 中心位于 `B_hold`
    
- bearing 允许中等变化，但不能达到 `loiter` 的环绕水平
    

---

##### 4) `pseudo_hover_racetrack`

固定翼的主要“悬停替代”风格。

enter_hold_center -> local_racetrack_hold

约束：

- racetrack 的几何中心位于 `B_hold`
    
- 局部转弯半径必须满足 `min_turn_radius`
    
- 全局 `encircle_cycles` 不得达到 `loiter` 水平
    

---

### 10.4 `loiter`

#### 公共定义

核心是：

- 在 `B_loiter` 内高占比活动
    
- 切向主导
    
- bearing 累计变化大
    
- 不允许持续转化成 `attack` 或 `retreat`
    

#### 风格

##### 1) `circle_loiter`

enter_band -> orbit

参数：

- `v_theta_cmd = const`
    
- `v_r_cmd = k_r * (r_target - r)`
    

---

##### 2) `ellipse_loiter`

enter_band -> ellipse_orbit

参数：

- `r_target(t) = r0 + A_r * sin(...)`
    
- `v_theta_cmd` 基本恒定
    

---

##### 3) `petal_loiter`

enter_band -> petal_orbit

参数：

- 半径做周期性收缩/扩张
    
- bearing 累计变化高
    
- 可以形成双瓣/花瓣式环绕，但语义仍是围站机动
    

---

##### 4) `racetrack_loiter`

enter_band -> tangential_leg + turn + tangential_leg + turn ...

适合固定翼。

---

##### 5) `probe_loiter`

orbit -> inward_probe -> orbit

参数：

- 某一时间段短暂向内试探
    
- 之后回到 `B_loiter`
    
- 不能持续向 `attack` 演化
    

硬约束：

- `encircle_cycles >= 0.75`
    
- `tangential_dominance >= 1.5`
    
- `close_score` 不得越过 `attack` 的主阈值
    

---

## 11. 机体航向 `yaw` 生成规则

当前系统里 `yaw` 主要是固定、朝原点或速度切线，这不足以表达“监视/攻击/诱骗”的指向语义。

建议改成阶段化 `yaw_mode`：

course_locked        # 朝速度方向  
station_facing       # 朝站点  
station_scan         # 围绕站点方向小范围扫描  
decoupled            # 与速度方向解耦，但受 yaw_rate 限制

### 推荐映射

- `attack.direct_commit`: `course_locked`
    
- `attack.probe_recommit`: `station_facing` 或 `station_scan`
    
- `hover`: `station_facing` / `station_scan`
    
- `loiter`: `station_facing` 或在站点附近扫描
    
- `retreat`: `course_locked` 或弱解耦
    

这样 `point_score` 才真正有意义。

## 12. 动力学展开与运动学检查

---

### 12.1 必须分两类动力学模型

#### A. `VelocityTrackingModel`

适用于：

- 多旋翼
    
- VTOL 的 hover/低速模式
    

更新方式：

a_cmd   = sat((v_des - v_cur) / tau, a_max)  
v_next  = sat(v_cur + a_cmd * dt, v_max)  
p_next  = p_cur + v_next * dt  
yaw_next= yaw_cur + sat(wrap(yaw_des - yaw_cur), yaw_rate_max * dt)

---

#### B. `CourseSpeedModel`

适用于：

- 固定翼
    
- VTOL 的巡航模式
    

更新方式：

speed_cmd = clip(||v_des||, v_min, v_dash)  
chi_des   = atan2(v_des.y, v_des.x)  
  
chi_next  = chi_cur + sat(wrap(chi_des - chi_cur), turn_rate_max * dt)  
speed_next= clip(speed_cur + sat(speed_cmd - speed_cur, a_long_limits) * dt,  
                 v_min, v_dash)  
  
v_xy_next = speed_next * [cos(chi_next), sin(chi_next)]  
vz_next   = clip(vz_cur + sat(vz_cmd - vz_cur, a_z_limits) * dt,  
                 -descent_rate_max, climb_rate_max)

---

### 12.2 必须检查的硬约束

对每条轨迹，至少检查：

1. 空间边界
    
2. 水平/三维最大速度
    
3. 最小速度（固定翼）
    
4. 纵向加速度 `a_long`
    
5. 横向加速度 `a_lat`
    
6. jerk
    
7. 爬升率 / 下降率
    
8. 最大 yaw_rate
    
9. 最大 turn_rate
    
10. 最小转弯半径（固定翼）
    
11. 时间连续性、无跳变
    

#### 推荐计算

u_v    = v_xy / ||v_xy||  
a_long = dot(a_xy, u_v)  
a_lat  = ||a_xy - a_long * u_v||  
curv   = |vx * ay - vy * ax| / (vx^2 + vy^2)^(3/2)  
R_turn = 1 / max(curv, eps)

---

### 12.3 检查顺序

#### 在线约束

在 rollout 时就做：

- speed clamp
    
- accel clamp
    
- yaw_rate clamp
    
- climb/descent clamp
    
- fixed-wing min speed clamp
    

#### 离线验证

生成完成后再做：

- 全轨迹 hard constraints
    
- 风险向量计算
    
- intent score 检查
    
- 模糊性排斥
    
- 多样性过滤
    

---

### 12.4 修复优先于直接丢弃

建议加入一次轻量修复：

1. 拉长阶段持续时间
    
2. 降低摆动振幅
    
3. 对 `v_cmd` 低通平滑
    
4. 降低末段冲击速度
    

修复一次仍失败再重采样。

---

## 13. 多样性控制

只做随机采样仍然会过拟合。必须加入**覆盖控制**。

### 13.1 样本特征向量

对每条候选样本计算：

f = [  
  duration_norm,  
  start_r_norm,  
  min_r_norm,  
  mean_speed_norm,  
  max_speed_norm,  
  altitude_excursion_norm,  
  path_ratio,  
  encircle_cycles_norm,  
  close_score,  
  dwell_score,  
  encircle_score,  
  point_score,  
  uncertain_score,  
  disengage_score,  
  abort_count_norm,  
  airframe_id,  
  style_id  
]

### 13.2 两层防重复

#### A. 语义覆盖分桶

至少按这些维度做直方图配额：

- `min_range`
    
- `mean_speed_norm`
    
- `uncertain_score`
    
- `duration`
    
- `encircle_cycles`
    

#### B. 最近邻距离过滤

在同 `intent + airframe + style` 桶内，若新样本与已有样本在 `f` 空间距离过小，则拒绝。

---

### 13.3 采样方式

参数采样建议用：

- Latin Hypercube
    
- Sobol
    
- 或分层均匀采样
    

不要只做独立均匀随机。

## 17. 生成主流程伪代码

for intent in intent_quota:  
    while accepted[intent] < quota[intent]:  
        airframe = sample_airframe(intent, cfg, rng)  
        semantic_target = sample_semantic_target(intent, cfg, rng)  
        style = select_style(intent, airframe, semantic_target, cfg, rng)  
        plan = build_stage_plan(intent, style, airframe, semantic_target, cfg, rng)  
  
        traj = rollout(plan, airframe, dt, cfg)  
  
        traj = maybe_repair(traj, airframe, cfg)  
  
        metrics = compute_station_metrics(traj, bands=cfg.bands, cfg=cfg)  
        risk_vector = compute_risk_vector(metrics, cfg)  
        scores = score_intents(risk_vector, metrics, cfg)  
  
        result = validate_sample(  
            traj=traj,  
            airframe=airframe,  
            target_intent=intent,  
            target_style=style,  
            scores=scores,  
            risk_vector=risk_vector,  
            cfg=cfg  
        )  
  
        if not result.ok:  
            log_failure(...)  
            continue  
  
        if not diversity_filter.accept(traj, risk_vector, airframe, style):  
            log_failure(reason="diversity_duplicate")  
            continue  
  
        save_sample(traj, metadata={  
            "intent": intent,  
            "style": style,  
            "airframe": airframe.name,  
            "risk_vector": risk_vector,  
            "scores": scores,  
            "stage_plan": plan  
        })

## 18. 与当前实现的迁移关系
基于你上传的当前规则，最自然的迁移方式是：

straight_penetration.direct_closing  
  -> attack.direct_commit  
  
non_straight_penetration.weave_approach  
  -> attack.weave_commit  
  
non_straight_penetration.climb_then_dive  
  -> attack.climb_dive_commit  
  
non_straight_penetration.turn_then_dive  
  -> attack.turn_dive_commit  
  
non_straight_penetration.zigzag_dive  
  -> attack.zigzag_commit  
  
hover.standoff_hover  
  -> hover.drift_hold / hover.true_hover 的基础实现  
  
loiter.surveillance_loiter  
  -> loiter.circle_loiter / loiter.probe_loiter 的基础实现  
  
retreat.direct_escape  
  -> retreat.direct_breakaway  
  
retreat.arc_escape  
  -> retreat.arc_breakaway  
  
retreat.zigzag_escape  
  -> retreat.zigzag_breakaway  
  
retreat.climb_escape  
  -> retreat.climb_breakaway

### 迁移时必须做的三件事

1. 把原先只给 `hover/loiter` 用的站点相对指标计算，推广到四类意图全量使用
    
2. 把原先“按意图固定”的速度/加速度/偏航率约束，改成“按机型 + 按阶段”
    
3. 把原先 `variant` 的意义从“几何子类”改成“同意图下的行为风格”

# 三、最终建议

这次修改的核心，不是“再补几种轨迹形状”，而是把生成器的中心从：

intent -> geometric path template

改成：

intent -> target risk region -> style -> stage commands -> airframe-constrained rollout

这样得到的才不是“飞行动作生成器”，而是你真正要的：

相对站点的风险行为生成器

并且这个生成器输出的不只是轨迹，还同时输出：

- 主意图
    
- 行为风格
    
- 机型约束
    
- 六维风险语义
    
- 动力学验证结果
