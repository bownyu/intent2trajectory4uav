# 2026-03-06 工程现状与轨迹增强计划

## 当前工程现状

当前仓库实现的是“意图 -> 运动学轨迹”的第一阶段能力：

- 支持五类意图的配置化轨迹生成；
- 支持按 `class_quota` 控制各意图样本数；
- 输出统一结构的轨迹 CSV 与 `metadata.csv`；
- 提供 GUI 播放器用于三维轨迹检查；
- `act_*` 仍为仿真回填预留字段。

尚未实现：

- PX4 / JSBSim 闭环仿真；
- `.ulg` 日志解析；
- `act_*` 实际轨迹回填；
- 基于仿真结果的真实飞行动力学特征校正。

## 当前主要问题

1. `non_straight_penetration` 轨迹类型单一，主要表现为规则化蛇形逼近，缺少更明显的爬升、转弯、俯冲等突防机动。
2. `loiter` 虽然能生成圆轨迹，但“盘旋”语义仍偏单一，缺少椭圆、8 字、偏心绕飞等巡查/观察类形态。
3. 关键意图的验证规则偏粗，只看首末距离或普通 yaw 标准差，无法充分保证标签语义。

## 本次增强目标

- 为 `non_straight_penetration` 引入配置驱动的多子类型生成；
- 为 `loiter` 引入配置驱动的多子类型生成；
- 强化轨迹几何验证与 yaw 统计方法；
- 保留现有对 `class_quota` 的样本数量控制方式；
- 在元数据中记录具体子类型，便于排查和可视化筛选。

## 计划中的子类型

### non_straight_penetration

- `weave_approach`
- `climb_then_dive`
- `turn_then_dive`
- `zigzag_dive`

### loiter

- `circle_hold`
- `ellipse_hold`
- `figure8_hold`
- `offset_orbit`

## 设计原则

- 所有固定形状都必须带随机尺度、随机方向、随机相位和随机持续时间；
- 生成行为必须可复现；
- 校验逻辑必须围绕“意图语义是否成立”展开，而不是只看弱特征；
- 配置优先，避免把分布控制写死在代码中。

## 实施后需要再次确认的事项

- 各子类型权重是否符合数据集分布预期；
- 非直线突防中爬升/俯冲的幅度是否足够明显；
- 盘旋类半径和轴长随机范围是否满足“大圆/小圆”多样性要求；
- 可视化观察下是否仍有“看起来不同但特征不显著”的样本。
